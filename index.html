Latest Updates

What changed recently

Project status and behavior updates reflected across the docs and runtime flow.

Card Template Enforced

`prompt.md` Step 4 now generates `card.json` directly with a strict Adaptive Card template: accent header, per-bullet TextBlocks, emphasis sources, and positive-style action button.

Covered Stories Dedup

`logs/covered-stories.txt` tracks all previously covered headlines. Bootstraps from last 10 Notion pages on first run. 30-day auto-trim prevents unbounded growth.

Opus Model + No Budget Cap

Both entry scripts now use `--model opus` with no `--max-budget-usd` cap. Webhook URL is refreshed from registry on every run to avoid stale env vars.

Single Delivery Guaranteed

Prompt explicitly forbids Claude from posting to webhooks. Only the calling script delivers via `notify-teams.ps1`/`.sh`, preventing duplicate Teams messages.

Teams

Adaptive Card delivery path

Current notifier scripts post a prebuilt card payload file directly to the webhook.

flowchart LR
    A[Claude writes logs/YYYY-MM-DD-card.json] --> B[notify-teams.sh / notify-teams.ps1]
    B --> C{JSON valid?}
    C -->|No| D[Fail fast]
    C -->|Yes| E[POST payload to webhook]
    E --> F[Teams channel message]
set webhook + test (macOS/Linux) 
# Set webhook URL
export AI_BRIEFING_TEAMS_WEBHOOK="<webhook-url>"

# Send today's default card file
bash scripts/notify-teams.sh

# Or send explicit file
bash scripts/notify-teams.sh --card-file logs/2026-03-17-card.json
set webhook + test (Windows PowerShell) 
# Persist webhook URL
[Environment]::SetEnvironmentVariable("AI_BRIEFING_TEAMS_WEBHOOK", "<webhook-url>", "User")

# Send today's default card file
.\scripts\notify-teams.ps1

# Or send explicit file
.\scripts\notify-teams.ps1 -CardFile ".\logs\2026-03-17-card.json"

Legacy note: `scripts/build-teams-card.py` remains in the repo for historical parser-based flow, but it is not invoked by default in current entry scripts.
